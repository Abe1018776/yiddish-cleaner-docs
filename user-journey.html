01 The Complete User Journey

Five phases from raw audio to published, fine-tuned Yiddish ASR. Click each phase to expand details.

1

Phase 1 MAPPING

Audio files and manual transcripts exist separately. Every transcript must be affiliated to an audio file. The mapping interface provides smart suggestions, search, and manual linking.

Mapping Interface Smart Suggestions First 15 Words Preview Hebrew Date Parsing CSV Export Fix Audio Length Auto-Update Suggestions Edit ALL Mappings Many-to-Many Splits Singing Detection Silence Detection

Mapping Interface

  • Unassigned transcripts column with algorithmic suggestions (date + keyword scoring)
  • Confidence scores displayed (0.0-1.0); green for scores above 0.5
  • Full-screen search modal across transcripts or audio (up to 100 results)
  • Six statistics cards: total audio, matched, unmatched, transcripts
  • Four tab navigation: 50-Hour Selection, All Matched, Unmatched Audio, Unmatched Transcripts
  • Filter by Year, Month, Content Type; free-text search

Mapping Table View

  • All audio displayed with Hebrew year/date (5711-5752)
  • Sortable columns with localeCompare for Hebrew text
  • Transcript count per audio (clickable to expand)
  • Pipeline status indicators per audio
  • Editable linking for ALL mappings

Needed Improvements

  • Fix audio length display (currently inaccurate)
  • Auto-update mapping suggestions on new data
  • Many-to-many relationships (split transcript across audio / split audio across transcripts)
  • Range selection for partial audio segments
  • Singing detection (flag non-speech audio segments)
  • Silence detection (flag dead air)
2

Phase 2 TRAINING PREP

50 hours selected with even distribution across years, months, and lecture types (Sicha/Maamar). Audio passes through a cleaning and review pipeline before training.

50-Hour Selection Script Cleaning (99%) 8 Cleaning Processors LLM Cleaning Alt Diff Viewer Karaoke Mode JEM Review Interface Confidence Scoring Bulk Approve Inline Editing
Click to expand details

Pipeline Stages

  • Needs Cleaning: raw transcripts with editorial content
  • Cleaned / Waiting Review: 8 processors remove non-spoken content
  • Approved for Training: JEM reviewer signs off
  • Sent for Training: locked, dispatched to fine-tuning pipeline

Automated Cleaning (Built)

  • SpecialChars: zero-width characters, smart quotes
  • SeifMarker: Hebrew section markers
  • TitleStyle: heading detection
  • Brackets: editorial annotations [...]
  • EditorialHebrew: non-spoken scripture references
  • Parentheses: editorial notes (...)
  • ForceRemove: pattern matching removal
  • Whitespace: normalization
  • Cleaning presets: minimal, titles_only, full_clean, with_editorial, heavy
  • LLM alternative: 5 AI providers with Yiddish-specific prompt

JEM Review Interface (Needed)

  • Diff Viewer: side-by-side old vs cleaned text
  • Karaoke Mode: word-level timestamps with audio playback
  • Confidence Scoring via stable-ts integration
  • Click low-confidence word to jump to problem in audio
  • Bulk select/deselect changes
  • Inline editing of individual words
  • Approve-all button with confirmation

Critical Rule

  • Training CANNOT use benchmark audio (enforced system-wide)
  • Pipeline stage indicators visible in mapping table

05 Immediate Priorities

The top 10 items that need to be addressed next, ranked by urgency and impact.

1

Fix Audio Length Display

Audio duration shown incorrectly in mapping interface. Quick fix with high visibility impact.

2

Auto-Update Mapping Suggestions

Suggestions should refresh automatically when new data is added instead of requiring redeployment.

3

Enable Editing ALL Mappings

Currently only unmatched items can be edited. Need to allow re-linking of already-matched pairs.

4

Singing / Silence Detection

Automatically flag audio segments containing singing (non-speech) or silence (dead air) to avoid training on bad data.

5

Many-to-Many Splits

Support splitting a transcript across multiple audio files, and vice versa. Essential for lectures split across tapes.

6

Add 2 More Benchmark Files

Expand gold standard from 5 to 7 files for more robust model evaluation across diverse content.

7

Build JEM Review Interface

Combine Diff Viewer + Karaoke Mode + Confidence Scoring into a single review screen for JEM staff.

8

Bulk Approve / Inline Edit

Allow reviewers to approve all changes at once or edit individual words inline without leaving the review screen.

9

Pipeline Stage Indicators

Show visual status (Needs Cleaning / Reviewed / Approved / Sent) for each audio file in the mapping table.

10

Enforce Benchmark != Training

System-level rule preventing benchmark audio from being used in the training pipeline. Critical for evaluation integrity.

07 UX Design Principles

Every interface in the platform follows these core principles for an intuitive, efficient experience.

◉

Very Clean Interface

Minimal chrome, maximum content. Dark theme with high contrast reduces eye strain during long review sessions. Every pixel serves a purpose.

⚡

Easy to Use

No training required. Smart defaults, contextual actions, and progressive disclosure. Complex features hidden until needed.

➜

Straightforward Workflow

Linear progression through phases. Each step clearly leads to the next. Status indicators show exactly where every item stands.

◆

Visual & Intuitive

Color-coded status, confidence scores as progress bars, waveform visualizations, karaoke word highlighting. Information conveyed visually first.

○

Minimal Cognitive Load

One primary action per screen. Batch operations for efficiency. Smart suggestions reduce decision fatigue. RTL text handled natively.
