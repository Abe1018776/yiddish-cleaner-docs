3 Color Meanings — Answering "Orange vs Red"

Color Confidence Range What It Means Action Needed
Green 80–100% Whisper is highly confident the word was correctly recognized and aligned to the right moment in the audio Trust it
Orange 40–79% Whisper recognized the word but isn't fully sure — could be background noise, unclear speech, or the word sounds similar to another Spot-check
Red Below 40% Whisper is guessing — the audio may be garbled, the speaker may be mumbling, or the alignment drifted. The backend flags these as "confusion zones" Review manually

The 0.4 threshold matches the backend's good_seg_prob_threshold — words the alignment engine internally considers "bad" will appear red in the UI. The 0.8 threshold separates "high confidence" from "needs a glance."

8 What Stays the Same

Audio Playback

Click any word to play from that position. The playWord() function and timeupdate listener are unchanged — they use the same start/end times stored in data attributes.

RTL Support

The .word-grid already uses direction: rtl. Background colors and borders are direction-independent. No RTL issues.

Backend API

Zero changes. Both the Flask endpoint and RunPod handler already return per-word confidence. The Cloudflare proxy passes it through unchanged.

Cleaning UI

The Flask transcript cleaner (templates/index.html) is a separate feature and is not affected by these changes.

Featureivrit.ai (Original)yiddish-audio-alignment (Extended)

Both the original and extended versions use w.probability from stable_whisper. The confidence score mechanism is inherited directly from the ivrit.ai foundation — the yiddish project extends it with smarter failure handling but the core scoring is identical.
