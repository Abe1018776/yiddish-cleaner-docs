Cleaner ↔ Mapping

The Mapping tool exports verified audio-transcript pairs as CSV. The Cleaner imports these via Google Sheets, pulling audio and transcript files from Drive into the training pipeline.

  • Mapping identifies which audio matches which transcript
  • Cleaner consumes matched pairs for cleaning and benchmarking
  • 423 curated pairs form the 50-hour training selection

Cleaner ↔ Alignment

Once transcripts are cleaned and validated in the Cleaner, they are sent alongside audio to the Alignment engine for word-level timestamp generation.

  • Cleaner triggers alignment jobs via API proxy
  • Alignment returns word-level timestamps and confidence scores
  • Results power the word-level review and custom WER interface

Mapping ↔ Alignment

Mapping provides the matched pairs; Alignment operates on them. Currently independent, but the unified system will chain them: match first, then align.

  • Mapping supplies audio + transcript pairs
  • Alignment processes pairs into timestamped output
  • Future: automated pipeline from match to alignment

Pipeline Support Systems

Implemented

Text Cleaning Engine

8 sequential processors handle Yiddish religious text normalization:

  • SpecialChars -- zero-width chars, smart quotes
  • SeifMarker -- Hebrew section markers
  • TitleStyle -- headings by style heuristics
  • Brackets -- inline editorial [annotations]
  • EditorialHebrew -- non-spoken scripture citations
  • Parentheses -- (inaudible) markers
  • ForceRemove -- user-configured patterns
  • Whitespace -- spacing normalization

5 presets: minimal, titles_only, full_clean, with_editorial, heavy. Alternative: LLM-based cleaning via Claude, GPT, or Groq.

Implemented

Clean Rate Scoring

Quality metric (0-100) evaluating how well cleaning was performed:

  • 9 penalty rules based on removal types
  • Global penalties for >50% text reduction
  • Global penalties for <50 words remaining
  • Ranges: Excellent (90-100), Good (75-89), Moderate (50-74), Low (25-49), Poor (0-24)
Implemented

WER / CER / Custom WER

Three accuracy metrics for ASR evaluation:

  • WER -- word error rate via Levenshtein distance with Hebrew nikkud normalization
  • CER -- character-level error rate
  • Custom WER -- only counts "critical errors" (true mistakes), ignoring acceptable Yiddish spelling variants

Formula: Custom WER = (insertions + deletions + critical_replacements) / total_reference_words

Implemented

Confidence Matching

Hybrid algorithm for audio-transcript pairing:

  • Date matching -- Hebrew calendar year, month, day comparison
  • Keyword matching -- content type (Sicha, Maamar, Farbrengen)
  • Scores range 0.0 to 1.0; amber below 0.5, green above
  • Multiple ranked suggestions per unmatched item
  • Human verification with one-click accept
Implemented

Confusion-Aware Alignment

Iterative algorithm for long-audio word-level alignment:

  • 120s sliding window with 30s hop monitors quality
  • Flags confusion zones when low-confidence ratio > 0.8
  • BreakableAligner extends stable-ts Aligner class
  • Recovery: backtrack to high-confidence segment, retry (max 2)
  • Stitches segments with continuous timestamps
Implemented

Karaoke Playback

Real-time word-level audio synchronization:

  • HTML5 audio timeupdate drives word highlighting
  • Click any word chip to seek to that timestamp
  • Color coding: green (≥0.5), yellow (≥0.2), red (<0.2) confidence
  • Pure JS implementation, zero dependencies

Audio File Formats

Supported input: MP3, WAV, M4A, OGG, FLAC. Audio is stored via Spatie Media Library in the Cleaner, sent as base64 to the Alignment engine, and referenced by Google Drive shareable link in the Mapping tool.

Transcript Formats

Source documents are DOCX/DOC/TXT, parsed via PhpWord in the Cleaner. The Mapping tool stores 15-word Hebrew first-line previews. The Alignment engine receives plain text (RTL encoded).

Alignment Output

Dual format: hierarchical segments (nested word objects with timestamps) and flat timestamp array. Each word includes: word, start, end, confidence. Model metadata and language designation included.

Mapping JSON

Static 246KB JSON with 9 collections: allAudio[], allTranscripts[], selected[], matched[], unmatchedAudio[] (with suggestedMatches[]), unmatchedTranscripts[], and stats. Loads in under 500ms.

06 API & Communication Layer

How frontends communicate with backends and how processing is triggered across systems.

Implemented

Cleaner: Inertia.js Pattern

No traditional REST API. Laravel controllers return Inertia::render('Page', $props). First load serves full HTML with embedded props; subsequent navigations fetch only JSON. Form submissions via router.post().

  • 65+ routes in web.php
  • 10 Inertia controllers for page rendering
  • 3 JSON API controllers (ASR, LLM, Words)
  • Form request validation classes
Implemented

Cleaner: WebSocket Events

Laravel Reverb broadcasts real-time events to the Vue frontend via Laravel Echo:

  • AudioSampleProcessed -- ASR job complete
  • BatchCompleted -- import batch done
  • DocumentProcessed -- transcript parsed
  • UI updates reactively without page refresh
Implemented

Alignment: API Proxy

Cloudflare Pages Function (align.js, 122 lines) acts as API gateway:

// Production endpoint POST https://align.kohnai.ai/api/align // Request body { "mode": "align" | "transcribe", "audio_base64": string, "audio_format": ".mp3", "text": string, // align mode only "language": "yi" } // Response { "full_text": string, "segments": [{ start, end, text, words: [...] }], "timestamps": [{ word, start, end, confidence }], "model": "yi-whisper-large-v3-turbo-ct2" }
Implemented

Alignment: Flask Dev Server

Development endpoints for local testing:

POST /api/alignment/align POST /api/alignment/transcribe GET /api/alignment/health

Flask Blueprint routes to either local processing or RunPod backend depending on configuration.

Implemented

External ASR APIs

Three ASR drivers abstracted behind AsrManager factory:

  • YiddishLabsDriver -- async API with polling, Yiddish-specific
  • WhisperDriver -- OpenAI Whisper API, segment timestamps
  • LocalWhisperDriver -- local Python via shell script
Implemented

External LLM APIs

Five LLM providers for AI-based transcript cleaning, managed by LlmManager:

  • OpenRouter, Anthropic (Claude), OpenAI (GPT)
  • Google AI (Gemini), Groq
  • Yiddish-specific cleaning prompts preserve original wording
  • Encrypted API keys stored per-user in ApiCredential model
